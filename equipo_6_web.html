Detección Inteligente de Fraude

Solución desarrollada por el Equipo 6 para el Challenge Final del curso Bedu Santander. IA accesible para proteger activos financieros y la confianza del cliente.

Contexto del Problema

El fraude financiero es un problema global que afecta directamente la rentabilidad, reputación y confianza en instituciones bancarias. Se estima que solo en 2023, las pérdidas por fraude en tarjetas superaron los 35 mil millones de dólares. Esto implica una necesidad urgente de soluciones inteligentes, automatizadas y replicables que mitiguen este riesgo a gran escala. Nuestro objetivo es contribuir a resolver este problema usando tecnologías accesibles, sin requerir programación avanzada.

Objetivos del Proyecto

  • Detectar fraudes bancarios en tiempo real, incluso en conjuntos de datos desbalanceados.
  • Reducir pérdidas económicas y mejorar la capacidad de reacción de las instituciones financieras.
  • Aplicar inteligencia artificial de forma ética, accesible y comprensible para usuarios no técnicos.
  • Utilizar plataformas no-code como KNIME para democratizar el uso de modelos predictivos.
  • Promover una cultura basada en datos para la toma de decisiones.

Flujo de Trabajo en KNIME

KNIME fue elegido por ser una herramienta visual, intuitiva y potente para crear flujos de trabajo analíticos sin necesidad de escribir código. Nuestro workflow se dividió en las siguientes fases:

  • 1. Preparación de datos: Se importaron 284,807 registros de transacciones. Se filtraron columnas irrelevantes, se verificó la integridad de los datos y se identificaron variables clave como Amount, V1 y V28, asociadas con patrones anómalos.
  • 2. Exploración de datos: Se utilizaron nodos de visualización para detectar outliers y comprobar la proporción de clases. La clase "Fraude" representaba solo el 0.172% de los datos, lo que evidenciaba un grave desbalance.
  • 3. Preprocesamiento: Se aplicó normalización min-max a las variables numéricas, y se utilizó una técnica de sobremuestreo (SMOTE) para equilibrar la proporción entre clases. También se dividió el dataset en entrenamiento (70%) y prueba (30%).
  • 4. Modelado y evaluación: Se entrenaron dos modelos: Regresión Logística y Random Forest. Ambos se evaluaron con métricas como accuracy, precisión, recall y matriz de confusión. Aunque Random Forest logró mayor precisión general (99.9%), la Regresión Logística detectó más fraudes verdaderos, lo cual es prioritario en estos casos.
  • 5. Interpretación y despliegue: Finalmente, se generaron visualizaciones de resultados y un sistema de alertas simuladas para mostrar cómo se podría implementar el modelo en producción.

Resultados

Los modelos desarrollados mostraron un desempeño sobresaliente. El modelo de Random Forest obtuvo 99.9% de accuracy general, y la regresión logística alcanzó un recall superior al 95% para la clase fraude. Se redujo significativamente la cantidad de transacciones fraudulentas no detectadas, lo que representa una mejora directa en la seguridad financiera. Además, al utilizar KNIME, se logró que cualquier persona con conocimientos básicos pudiera replicar y adaptar esta solución a otras instituciones o contextos.

Futuro de la Solución

Planteamos varias mejoras para escalar esta solución:

  • Integrar variables contextuales como hora de transacción, ubicación IP o historial del cliente para enriquecer el modelo.
  • Implementar técnicas más avanzadas como XGBoost, redes neuronales profundas o modelos de atención.
  • Realizar validación cruzada k-fold para robustecer el desempeño en distintos escenarios y reducir el sobreajuste.
  • Desarrollar un dashboard en tiempo real para visualizar alertas, estadísticas de fraude y evolución del modelo.
  • Crear una API RESTful para que la solución pueda integrarse con plataformas bancarias existentes.
